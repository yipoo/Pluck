# PRD — Pluck (Mac 端 OCR / 截图 / 剪贴板套件)

**版本**:0.1(MVP 立项)
**日期**:2026-04-26
**作者**:dinglei

---

## 1. 背景与定位

### 1.1 问题陈述

中国 Mac 用户(估 3000-4000 万,深度工作者 600 万)面对的"截图 + OCR + 剪贴板"工作流当前是碎片化的:

- **截图**:系统自带 ⌘⇧4 / ⌘⇧5(简陋)、Snipaste、Xnip、CleanShot(海外、上云)
- **OCR**:白描、Bob、PixPin、TextSniper、OwlOCR — 每个体验割裂
- **剪贴板**:Paste、Maccy、PopClip
- **翻译**:Bob、欧路、有道

用户痛点:
1. 装 5 款工具才能做完一个完整工作流(截图 → OCR → 翻译 → 贴到笔记)
2. 海外工具(CleanShot、Raycast)无中文友好,且数据上云
3. 律师 / 医生 / 政府 / 财务等人群处理敏感数据,**云端 OCR 不可用**
4. 国产 PixPin / 白描功能强但没整合 AI(智能选词、智能裁剪、智能命名)

### 1.2 产品定位

**Pluck = 全本地隐私 + 一站式工作流 + AI 增强**,做"PixPin 进阶版" + "国产 CleanShot"。

三句话差异化:
1. **全本地处理**:OCR / 翻译 / 标注全部本地完成,默认不联网,适合处理敏感文档
2. **一站式工作流**:截图 → OCR → 标注 → 翻译 → 复制 / 分享 / 保存,一个 App 全搞定
3. **AI 增强(可选)**:接入备案 LLM API(豆包 / Kimi / DeepSeek)做智能选词、智能命名、内容总结 — 用户可选择"完全本地"或"AI 增强"模式

### 1.3 不做什么(out of scope)

明确不做的事(避免 scope creep,避免合规/技术泥潭):
- ❌ 屏幕录制 / 录屏(留给 v3)
- ❌ 跨设备同步 / iCloud 同步(留给 v2)
- ❌ iOS 配套 App(留给 v2)
- ❌ 团队协作(留给 v3)
- ❌ Windows 版(留给 v2)
- ❌ 自建 LLM / 自训模型
- ❌ 上架 Mac App Store(v1 直接走自有官网下载)

## 2. 目标用户

### 2.1 主要用户画像

**P1 — 知识工作者(主要付费方)**
- 25-40 岁,Mac 重度用户
- 职业:产品经理 / 设计师 / 程序员 / 自由职业
- 痛点:每天截 10-30 次图,OCR 频繁,用过 PixPin / 白描 / Bob
- 付费意愿:¥98 买断 / ¥48 年订阅 接受度高(对标 Setapp ¥69/月、CleanShot $29 终身)

**P2 — 隐私敏感专业人士**
- 律师 / 医生 / 公务员 / 金融合规 / 翻译
- 痛点:工作内容敏感,公司禁止使用上云 OCR
- 付费意愿:¥298 终身版 + 团队授权

**P3 — 学生 / 内容创作者**
- 大学生 / 考研党 / 自媒体
- 痛点:文献 OCR、长截图、社媒素材整理
- 付费意愿:¥48/年 学生折扣

### 2.2 反向定义(不是我们的用户)

- 偶尔截图的轻量用户(免费版 Snipaste 足够)
- 团队协作主导用户(用 Slack / 飞书内置功能)
- Windows 用户(v1 不支持)

## 3. MVP 功能范围(v0.1.0)

> 原则:**砍到不能再砍**。MVP 跑通"截图 → OCR → 复制 → 剪贴板回看"主线,其他功能 v0.2+ 加。

### 3.1 必须有(P0)

| 功能 | 描述 | 验收标准 |
|------|------|---------|
| **F1 全局热键截图** | ⌃⌥A(可自定义)触发区域选择截图 | 1 秒内呼出选择框 |
| **F2 区域 OCR** | 截图后自动识别中英文 + 复制到剪贴板 | 中文识别准确率 ≥ 95%(印刷体);A4 尺寸 < 1 秒 |
| **F3 菜单栏图标** | macOS 顶栏常驻,点击展开历史 | 启动后常驻不闪退 |
| **F4 剪贴板历史** | 监听并保存最近 100 条剪贴板内容(文本 + 图片) | 重启不丢失;搜索 100 条 < 100ms |
| **F5 快速搜索** | 在历史面板内全文搜索 | 关键词高亮 |
| **F6 设置面板** | 热键 / 历史条数上限 / 启动项 / 隐私选项 | 即改即生效 |
| **F7 启动项** | 登录时自动启动 | macOS 14+ ServiceManagement 集成 |

### 3.2 应该有(P1)

| 功能 | 描述 |
|------|------|
| **F8 截图标注** | 矩形 / 箭头 / 文字 / 马赛克(像 Annotate / Skitch) |
| **F9 长截图** | 截网页 / 长文档(滚动拼接) |
| **F10 贴图** | 把截图钉在桌面上(像 Snipaste) |
| **F11 表格 OCR** | 识别表格结构,导出 markdown 表格 |
| **F12 多语言翻译** | OCR 结果一键翻译(本地词典 / 接 API) |
| **F13 自动命名** | 截图按内容智能命名(可选 AI 辅助) |

### 3.3 可以有(P2,看节奏)

| 功能 | 描述 |
|------|------|
| **F14 AI 增强模式** | 接入备案 LLM:智能总结 / 智能扩写 / 智能翻译(用户开关) |
| **F15 浮窗模式** | OCR 结果以浮窗显示,可拖动 |
| **F16 主题** | 浅色 / 深色 / 自动 |
| **F17 离线模型** | 高级 OCR 模型(PaddleOCR Lite / RapidOCR)做手写识别 |

### 3.4 v0.1.0 发布前必须的非功能性需求

- macOS 14.0+ 兼容
- 内存占用空闲态 < 100MB,工作态 < 300MB
- 启动时间 < 1.5 秒
- 应用签名 + 公证(notarization)
- 隐私清单(PrivacyInfo.xcprivacy)
- 中文界面(简体)+ 英文兜底
- 关于页面包含"完全离线运行"声明

## 4. 用户故事(关键 5 条)

### US-1:截图 OCR 一气呵成
> 作为产品经理,我看到一份扫描版需求文档,我希望按一个快捷键圈选区域,文字自动跑到剪贴板里,我立即粘贴到飞书。

**验收**:从按热键到剪贴板有内容 ≤ 2 秒(A4 半屏);成功率 ≥ 95%。

### US-2:剪贴板历史回查
> 作为开发者,我刚刚拷贝了 5 段代码片段,但被覆盖丢了第 2 段。我希望按 ⌃⌥V 看到历史并恢复任意一条。

**验收**:呼出历史 ≤ 200ms;历史最近 100 条不丢失。

### US-3:律师隐私场景
> 作为律师,我处理客户合同截图,**公司禁止使用任何上云 OCR**。我需要一个"完全离线"的选项,App 启动后我能在设置里关掉所有网络请求。

**验收**:开启"严格离线"后,App 不发起任何 outbound 网络;`Little Snitch` 验证零外联。

### US-4:长截图归档
> 作为设计师,我要把一个 Notion 长页面截下来归档。期望选择"长截图",自动滚动拼接,导出 PNG。

**验收**:能拼接 4-6 屏内容,无错位 / 撕裂;导出文件 < 5MB。

### US-5:历史搜索
> 作为内容创作者,我两周前 OCR 过一段话,关键词模糊记得有"调研"二字。希望在历史里搜到。

**验收**:搜索 1000 条历史 < 200ms,关键词高亮。

## 7. 产品命名

**已定名:Pluck**(2026-04-26)

选择理由:
- "to pluck" = 挑出 / 摘取 — 与产品动作(从图里 pluck 文字、从屏幕 pluck 选区、从历史里 pluck 片段)语义对齐
- 单音节,英文易记,中文营销可直接用英文(对标 Notion / Bear / Linear)
- 排除 Snap(与 Snap Inc 商标冲突)、Snip(Microsoft Snipping Tool 占用)、SnagIt(TechSmith)、Klippa(荷兰 OCR 公司)等已占品牌
- 排除候选:Pickly / Glint / Lume / Knack — 备用,若 .app 域名或商标受阻可启用

W7 公证前必须完成的命名相关事项:
1. **域名**:首选 `pluck.app`,备选 `pluck.cn` / `getpluck.com` / `pluckapp.com`,预算 ¥100-2000
2. **商标**:中国 Class 9(软件)+ Class 42(SaaS),约 ¥1500-3000 / 件;海外可用 LegalZoom / Trademarkia
3. **GitHub Org**:`pluck-app` 或 `getpluck`
